## 1) SALES_ORDER_ITEMS

- 역할: 수주/주문 원본
- 분석 관점: 수요, 납기 이슈 신호, 고객/품목별 매출 기여
- 주요 키:
  - 수주 식별: `수주번호`
  - 품목 식별: `품번`
  - 고객 식별: `거래처`, `사용처`
  - 날짜: `수주일`, `최초출하의뢰일`, `최종출하의뢰일`
- 주요 수치:
  - `수량`, `판매금액`, `매출금액`, `원화판매금액계`
  - `재고수량`, `구매요청수량`, `구매입고수량`, `매출수량`
- 원인 분석용 텍스트:
  - `비고`, `마스터비고`, `마스터메모`, `진행사유`
- 상태/분류:
  - `영업진행상태`, `구매진행상태`, `수주구분`
  - `품목분류1~5`, `품목대분류/중분류/소분류`

## 2) INVOICE_ITEMS

- 역할: 출고/송장 원본
- 분석 관점: 출고 실적, 매출 확정, 미매출/반품 상태
- 주요 키:
  - 송장 식별: `거래명세서번호`
  - 수주 연결: `수주번호`
  - 품목 식별: `품번`
  - 고객 식별: `거래처`, `청구처`, `납품거래처`
  - 날짜: `거래명세서일`, `최근세금계산서일`
- 주요 수치:
  - `수량`, `매출수량`, `판매금액`, `매출금액계`, `원화판매금액계`
  - `미매출수량`, `미매출금액`
- 원인 분석용 텍스트:
  - `비고`, `마스터비고`, `마스터메모`
- 상태/분류:
  - `진행상태`, `출고구분`, `통화`, `환율`
  - `품목분류1~4`, `품목대분류/중분류/소분류`

## 3) PURCHASE_REQUEST_ITEMS

- 역할: 구매요청 원본
- 분석 관점: 조달 수요, 리드타임 위험, 요청 대비 발주 진행
- 주요 키:
  - 구매요청 식별: `구매요청번호`
  - 수주 연결: `수주번호`
  - 품목 식별: `품번`
  - 고객/거래처: `거래처`, `수주거래처`
  - 날짜: `요청일`, `요청납기일`, `발주예정일`
- 주요 수치:
  - `구매요청수량`, `품의진행수량`, `발주진행수량`, `조달일수`
- 원인 분석용 텍스트:
  - `비고`, `비고2`, `중단사유`
- 상태/분류:
  - `진행상태`, `중단`, `중단요청`, `직배송`
  - `Maker`, `품목분류1~4`

## 4) 권장 조인 기준

1. 수주 ↔ 송장
- 기본: `수주번호` + `품번`
- 보조: `거래처`, `품명`

2. 수주 ↔ 구매요청
- 기본: `수주번호` + `품번`
- 보조: `거래처`, `요청일/수주일` 근접성

3. 송장 ↔ 구매요청
- 기본: `수주번호` + `품번`

## 5) 데이터 타입 처리 원칙

- CSV 숫자 컬럼은 문자열일 수 있으므로 집계 전 변환한다.
  - 예: `TRY_CAST(NULLIF("매출금액", '') AS DECIMAL(18,2))`
- 날짜 컬럼도 문자열일 수 있으므로 비교 전 변환한다.
  - 예: `TRY_CAST("수주일" AS DATE)`
- 쉼표 포함 숫자 대응이 필요하면 문자열 처리 후 캐스팅한다.
  - 예: `TRY_CAST(REPLACE(CAST("매출금액" AS VARCHAR), ',', '') AS DECIMAL(18,2))`
