# Metric Definitions

version: 1.0
updated_at: 2026-03-23

## 1) 매출 관련

### 총매출
- 정의: 분석 대상 기간의 매출 합계
- 우선 컬럼:
  1. `INVOICE_ITEMS.원화판매금액계`
  2. `INVOICE_ITEMS.매출금액계`
  3. `SALES_ORDER_ITEMS.매출금액`
- 계산 예:
```sql
SUM(TRY_CAST(NULLIF("원화판매금액계", '') AS DECIMAL(18,2)))
```

### 거래처별 매출 TOP N
- 정의: 거래처별 총매출 내림차순 상위 N개
- 그룹 키: `거래처`

## 2) 납기/리드타임 관련

### 납기 지연 건수(실무형 근사)
- 정의: 비고/진행사유/메모에서 지연 패턴이 감지된 건수
- 패턴 예: `재고 없음`, `4~6주`, `지연`, `리드타임`
- 계산: 라벨링 후 `LONG_LEADTIME`, `STOCK_OUT` 집계

### 지연률
- 정의: 지연건수 / 전체 주문건수
- 분모: 동일 필터의 전체 주문 line count

### 평균 지연일수(데이터 있을 때)
- 전제: 주문일/출고일 또는 요청납기일/실제출고일이 모두 존재
- 계산: `AVG(date_diff('day', 기준일, 비교일))`

## 3) 재고/미충족 관련

### 미매출금액
- 정의: 아직 매출 확정되지 않은 금액
- 컬럼: `INVOICE_ITEMS.미매출금액`

### 미충족 수요(근사)
- 정의: 주문수량 - 출고수량
- 계산: `SALES_ORDER_ITEMS.수량 - INVOICE_ITEMS.매출수량`
- 조인 키: `수주번호 + 품번`

## 4) 구매/조달 관련

### 구매요청 집중도
- 정의: 특정 기간에 품목별 구매요청 건수/수량 비중
- 컬럼: `PURCHASE_REQUEST_ITEMS.구매요청수량`, `품번`, `품목분류1`

### 조달 리스크 신호
- 정의: `조달일수` 증가 + `중단/중단요청` + 지연 패턴 동시 발생

## 5) 텍스트 라벨 기반 리스크

라벨 소스: `03_note_codebook.md`

- `STOCK_OUT`: 재고 부족 신호
- `LONG_LEADTIME`: 리드타임 장기화
- `CUSTOMER_CHANGE`: 고객 요청/변경
- `DIRECT_SHIP`: 직배송/직납 이슈
- `DISCOUNT`: 가격/프로모션 조건
- `QUALITY_ISSUE`: 품질/불량/리콜

## 6) 기준 기간 정책

- 기본 최근 기간: 최근 90일
- 필요시 최근 3개월/6개월/12개월로 확장
- 데이터가 과거에 집중된 경우 절대기간을 우선 사용
  - 예: `BETWEEN DATE '2020-01-01' AND DATE '2020-12-31'`
