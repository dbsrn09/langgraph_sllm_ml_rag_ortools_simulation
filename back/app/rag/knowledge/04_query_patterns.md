# Query Patterns

version: 1.0
updated_at: 2026-03-23

## 패턴 A: 거래처/품목 매출 TOP

- 질문 예: 거래처별 매출 상위 10개
- 기본 테이블: `SALES_ORDER_ITEMS` 또는 `INVOICE_ITEMS`
- 핵심 컬럼: `거래처`, `품번`, `매출금액`, `원화판매금액계`
- 출력: TOP N 목록

## 패턴 B: 납기 지연 원인 분석

- 질문 예: 세종대 납기 지연의 주요 원인
- 기본 테이블:
  - `SALES_ORDER_ITEMS` (비고/진행사유, 수주일)
  - 필요시 `INVOICE_ITEMS` (출고/매출 관점)
- 처리:
  1) 지연/재고/요청 패턴 라벨링
  2) 라벨별 건수/비율 계산
  3) 고객/품목 TOP N 도출

## 패턴 C: 구매요청 집중 품목

- 질문 예: 가장 많이 구매요청된 품목분류1
- 기본 테이블: `PURCHASE_REQUEST_ITEMS`
- 핵심 컬럼: `요청일`, `품목분류1`, `구매요청수량`, `거래처`

## 패턴 D: 미매출/미충족 분석

- 질문 예: 미매출금액이 큰 거래처
- 기본 테이블: `INVOICE_ITEMS`
- 핵심 컬럼: `미매출금액`, `미매출수량`, `거래처`

## 패턴 E: 교차 테이블 분석(수주↔송장)

- 질문 예: 수주 대비 출고 부족 품목
- 조인 키: `수주번호 + 품번`
- 기본 계산:
  - 수주수량 = `SALES_ORDER_ITEMS.수량`
  - 출고수량 = `INVOICE_ITEMS.매출수량` 또는 `수량`
  - 미충족 = 수주수량 - 출고수량

## SQL 작성/변환 규칙

- 가능한 경우 DuckDB 문법 직접 사용
- SQL Server 문법이 나와도 normalizer가 변환:
  - `TOP N` -> `LIMIT N`
  - `[컬럼]` -> `"컬럼"`
  - `TRY_CONVERT` -> `TRY_CAST`
  - `ISNULL` -> `COALESCE`
  - `GETDATE()` -> `current_date`

## 실패 시 점검 순서

1. 날짜 필터가 현재 날짜 기준으로 과도한지 확인
2. 금액/수량 컬럼 캐스팅 확인
3. 텍스트 함수 대상 컬럼 타입 확인
4. 조인 키 누락 여부 확인
5. 빈 결과면 기간/조건 완화 후 재시도
